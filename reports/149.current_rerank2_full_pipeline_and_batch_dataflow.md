# 当前 Rerank-2 完整框架与 Batch 数据流详解

## 核心目标 (Objective)

当前整个系统的目标是：

> 在不继续破坏 TTS 生成主干的前提下，提高 Visual TTS / dubbing 的音视频同步能力。

具体指标目标：

- 降低 `LSE-D`
- 提高 `LSE-C`
- 保持 `WER` 不明显变差
- 保持 `UTMOS / speaker similarity` 不明显变差

当前不再试图直接修改 `q0 / g0 / H`，而是采用：

```text
Frozen v14.6b Generator
        +
K-candidate generation
        +
Independent WavLM-AVSync scorer
        +
Candidate reranking
```

---

## 1. 为什么现在改成 Reranking 框架

### 1.1 前面失败的路线

前面的失败主要来自一个共同问题：

> 我们试图用 planner state 直接改 AR decoder 内部的 q0/g0 表示，但 LSE 本质上是整段音视频轨迹同步问题，不是单步 q0 token matching 问题。

具体失败：

| version | 操作位置 | 失败原因 |
|---|---|---|
| `v15.1a` | train-time q0/g0 adapter | 训练后主干分布被损坏，adapter-off 也无法恢复 |
| `v15.2a` | inference-time q0 logits bias | q0-token/viseme prior 推坏 acoustic token distribution |
| logmel sync scorer | candidate selector | val acc 接近随机，无法判断同步 |

因此当前最重要的转向是：

> 不再改生成过程本身，而是在生成多个候选后选择更好的候选。

### 1.2 为什么这个方向仍然有价值

mini20 / mini50 的 oracle 证明：

> frozen `v14.6b` 的候选空间里，确实已经存在同步更好的候选。

mini50 上 `cand0/cand1/cand2` 的 WER-constrained oracle：

| system | baseline WER | baseline LSE-D | baseline LSE-C | oracle WER | oracle LSE-D | oracle LSE-C |
|---|---:|---:|---:|---:|---:|---:|
| system1 oracle_lse_c | 3.2130 | 10.8100 | 3.3041 | 2.8130 | 10.3206 | 3.8109 |
| system2 oracle_lse_c | 5.8321 | 10.9713 | 3.1048 | 2.3607 | 10.3209 | 3.8728 |
| system1 oracle_lse_d | 3.2130 | 10.8100 | 3.3041 | 1.8130 | 10.3026 | 3.7690 |
| system2 oracle_lse_d | 5.8321 | 10.9713 | 3.1048 | 3.3607 | 10.2811 | 3.8064 |

这说明：

```text
生成器不是完全没有能力
问题是 sampling 后我们没有正确选择 candidate
```

## 2. 当前完整系统总览

当前框架分成两个阶段：

```text
Stage A: Candidate Generation
Stage B: Candidate Reranking
```

### 2.1 Stage A: Candidate Generation

使用 frozen `v14.6b epoch00`：

```text
model config = glo-var/v14.6b
ckpt = artifacts/Glo-Varv14/v6b/epoch00.ckpt
```

对每个 LRS3 test sample，生成多个候选：

```text
candidate_0.wav
candidate_1.wav
candidate_2.wav
...
```

这些候选来自不同 seed / top_p / temperature。

注意：

- `v14.6b` 不训练；
- TTS backbone 不更新；
- decoder、planner、codebook heads 都 frozen；
- 这里只负责生成候选。

### 2.2 Stage B: Candidate Reranking

训练一个独立 scorer：

```text
WavLM-AVSync Scorer
```

它输入：

```text
generated candidate audio
target video lip feature
```

输出：

```text
sync score
```

然后选择：

$$
k^* = \arg\max_k Score(A^{(k)}, V)
$$

最终输出：

```text
candidate_{k*}.wav / candidate_{k*}.mp4
```

## 3. Stage A: v14.6b Generator 数据流

### 3.1 输入

对一个 test sample：

- `Text_ref`: reference audio 对应文本
- `Text_tar`: target video 对应文本
- `Audio_ref`: reference audio
- `Video_tar`: target video / lip feature

经过预处理：

```text
Text_ref + Text_tar -> phoneme token sequence x
Audio_ref -> EnCodec tokens y_ref
Video_tar -> AV-HuBERT lip feature v
```

shape 近似：

$$
x \in \mathbb{R}^{T_x}
$$

$$
y_{ref} \in \mathbb{R}^{K \times T_{ref}}, \quad K=4
$$

$$
v \in \mathbb{R}^{T_v \times 1024}
$$

### 3.2 v14.6b 内部流程

简化流程：

```text
text tokens x
    -> text embedding

reference audio tokens y_ref
    -> delayed codebook pattern
    -> audio embedding

target lip feature v
    -> video tokenizer / projector
    -> planner
    -> plan_hidden

[text embedding, ref audio embedding, video/planner condition]
    -> AR decoder
    -> codebook heads
    -> q0/q1/q2/q3 tokens
    -> EnCodec decoder
    -> generated audio
```

更具体：

```text
for generation step t:
    decoder hidden H_t
        -> codebook routing / hierarchical conditioning
        -> g_t^(0), g_t^(1), g_t^(2), g_t^(3)
        -> q0_t, q1_t, q2_t, q3_t logits
        -> sample codebook tokens
        -> append sampled token embedding to AR history
```

### 3.3 当前 reranking 框架下的重要限制

在 Rerank-2 中：

```text
v14.6b generator 完全 frozen
```

不做：

- 不改 `H_t`
- 不改 `g_t`
- 不改 q0 logits
- 不加 planner adapter
- 不训练 decoder

只做：

```text
重复采样 K 次，得到 K 个候选
```

## 4. Stage B: WavLM-AVSync Scorer 训练数据流

这一部分是当前正在训练的模型。

### 4.1 一个训练 batch 的原始输入

训练数据来自真实 LRS3 train clip。

一个 batch 包含 `B` 个真实 audio-video pair：

```text
batch = {
    audio: real waveform windows,
    video: real lip feature windows,
    label: offset class
}
```

shape：

$$
A_{raw} \in \mathbb{R}^{B \times L}
$$

$$
V_{raw} \in \mathbb{R}^{B \times W \times 1024}
$$

$$
y_{\Delta} \in \{0,\dots,8\}^{B}
$$

当前参数：

```text
B = 8
W = 48 frames
offset classes = [-8,-6,-4,-2,0,2,4,6,8]
```

25Hz 下：

```text
1 frame = 640 audio samples at 16kHz
48 frames = 30720 audio samples
```

因此：

$$
L = 48 \times 640 = 30720
$$

### 4.2 Dataset 内部如何构造 offset label

对真实 clip：

```text
video feature V_real
audio waveform A_real
```

随机选择：

```text
start frame t
offset Δ ∈ [-8,-6,-4,-2,0,2,4,6,8]
```

video window：

$$
V_{win} = V_{real}[t : t+W]
$$

audio window：

$$
A_{win} = A_{real}[t+\Delta : t+\Delta+W]
$$

如果：

```text
Δ = 0
```

就是同步样本。

如果：

```text
Δ != 0
```

就是错位样本。

训练目标是让 scorer 判断这个 audio window 相对 video window 偏移了多少。

## 5. WavLM-AVSync Scorer 模型内部数据流

### 5.1 Audio Branch

输入 waveform：

$$
A_{raw} \in \mathbb{R}^{B \times 30720}
$$

经过 frozen WavLM：

$$
H_a = \text{WavLM}(A_{raw})
$$

输出 shape 约为：

$$
H_a \in \mathbb{R}^{B \times T_a \times 768}
$$

由于 WavLM 的 frame rate 不一定正好等于 25Hz，所以插值到 video window 长度：

$$
\tilde{H}_a = \text{Interpolate}(H_a, W)
$$

$$
\tilde{H}_a \in \mathbb{R}^{B \times W \times 768}
$$

再经过 audio projection：

$$
Z_a = \text{Proj}_a(\tilde{H}_a)
$$

$$
Z_a \in \mathbb{R}^{B \times W \times D}
$$

当前：

```text
D = 256
```

### 5.2 Video Branch

输入 AV-HuBERT lip feature：

$$
V_{raw} \in \mathbb{R}^{B \times W \times 1024}
$$

经过 video projection：

$$
Z_v = \text{Proj}_v(V_{raw})
$$

$$
Z_v \in \mathbb{R}^{B \times W \times D}
$$

### 5.3 Audio-Video Fusion

先归一化：

$$
\hat{Z}_a = \text{Normalize}(Z_a)
$$

$$
\hat{Z}_v = \text{Normalize}(Z_v)
$$

构造四类特征：

$$
F = [
\hat{Z}_a,\;
\hat{Z}_v,\;
|\hat{Z}_a-\hat{Z}_v|,\;
\hat{Z}_a \odot \hat{Z}_v
]
$$

shape：

$$
F \in \mathbb{R}^{B \times W \times 4D}
$$

即：

$$
F \in \mathbb{R}^{B \times 48 \times 1024}
$$

### 5.4 Temporal Scorer

把时间维送入 Conv1D：

```text
F
  -> Conv1D
  -> GELU
  -> Conv1D
  -> GELU
  -> mean pooling over time
```

得到：

$$
g \in \mathbb{R}^{B \times D}
$$

最后输出 offset logits：

$$
o = W_o g + b_o
$$

$$
o \in \mathbb{R}^{B \times 9}
$$

---

## 6. Scorer 损失函数

使用 cross entropy：

$$
\mathcal{L}_{offset}
=
CE(o, y_{\Delta})
$$

其中：

- $o$ 是 9 类 offset logits；
- $y_\Delta$ 是真实 offset class；
- class 0-8 对应：

```text
[-8,-6,-4,-2,0,2,4,6,8]
```

这个 loss 不使用：

- LSE-D
- LSE-C
- SyncNet final evaluator
- generated candidate oracle label

因此它是一个独立训练的 AV correspondence scorer。

## 8. Reranking 推理流程

训练完成后，使用 `best.pt` 或某个 checkpoint 对候选做 rerank。

### 8.1 输入

对每个 test sample：

```text
candidate_0.wav
candidate_1.wav
candidate_2.wav
target_lip_feature.npy
```

### 8.2 对每个候选打分

对候选 $k$：

$$
s_k = Score(A^{(k)}, V)
$$

当前 scorer 输出 offset logits：

$$
o^{(k)} \in \mathbb{R}^{9}
$$

可以用同步类别的 margin 作为分数：

$$
s_k =
p(\Delta=0)
-
\max_{\Delta \neq 0} p(\Delta)
$$

直觉：

- 如果 `offset=0` 概率最高，说明 audio/video 同步；
- 如果某个 shifted offset 概率更高，说明候选可能不同步。

### 8.3 候选选择

选择：

$$
k^* = \arg\max_k s_k
$$

最终输出：

```text
candidate_{k*}
```

### 8.4 后续需要加入的 guard

为了避免 scorer 选到 WER/quality 坏候选，后续应加：

- duration guard；
- silence/repetition guard；
- low-margin fallback to cand0；
- possibly ASR guard。

但第一阶段先验证 scorer 本身是否能恢复 oracle LSE gain。

## 9. 当前框架下一个 batch 的完整流动

下面以 scorer training batch 为例。

### Step 1: Dataset 采样

从 LRS3 train manifest 取 `B=8` 个样本：

```text
pair_id_1, pair_id_2, ..., pair_id_8
```

每个 pair_id 对应一个 target utterance：

```text
speaker_targetutt
```

读取：

```text
video_origin/speaker_targetutt.mp4
lip_feature/speaker_targetutt.npy
```

### Step 2: 构造 shifted pair

对每个样本随机选：

```text
start frame t
offset Δ
```

得到：

```text
video_window = video[t : t+48]
audio_window = audio[t+Δ : t+Δ+48]
label = class_index(Δ)
```

batch 输出：

```python
{
    "audio": [B, 30720],
    "video": [B, 48, 1024],
    "label": [B]
}
```

### Step 3: Audio through frozen WavLM

```text
audio [B, 30720]
    -> WavLM_BASE_PLUS
    -> audio_hidden [B, T_a, 768]
    -> interpolate to 48 frames
    -> audio_hidden [B, 48, 768]
```

### Step 4: Projection

```text
audio_hidden [B, 48, 768]
    -> audio_proj
    -> audio_z [B, 48, 256]

video [B, 48, 1024]
    -> video_proj
    -> video_z [B, 48, 256]
```

### Step 5: Fusion

```text
audio_z, video_z
    -> normalize
    -> concat(audio_z, video_z, |audio_z-video_z|, audio_z*video_z)
    -> fused [B, 48, 1024]
```

### Step 6: Temporal scorer

```text
fused [B, 48, 1024]
    -> Conv1D temporal scorer
    -> mean pool
    -> g [B, 256]
```

### Step 7: Offset prediction

```text
g [B, 256]
    -> linear head
    -> logits [B, 9]
```

### Step 8: Loss

```text
logits [B, 9]
label [B]
    -> cross entropy
    -> loss
```

### Step 9: Backprop

只更新：

- audio projection；
- video projection；
- temporal scorer；
- offset head。

不更新：

- WavLM；
- v14.6b generator；
- EnCodec；
- AV-HuBERT feature extractor。

## 10. 当前最重要的判断

现在整套框架的逻辑是：

```text
v14.6b 负责生成
WavLM-AVSync scorer 负责选择
```

不是：

```text
planner 负责改 q0
```

也不是：

```text
scorer 参与生成 token
```

这点非常关键。

Rerank-2 成功的条件是：

1. scorer 能在真实 shifted pair 上学会 offset；
2. scorer 能迁移到 generated candidates；
3. reranking 能改善 mini50 LSE；
4. 同时 WER 不变差。

目前已经完成的是：

```text
条件 1 初步成立：step500 val_acc = 0.495
```

后面还需要验证：

```text
条件 2-4
```

即训练完成后做 candidate reranking evaluation。
