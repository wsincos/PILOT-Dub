# PILOT-Dub Project Migration Summary

## 1. Goal

Migrate the final paper-oriented code path from the exploratory `Glo-Var` workspace into:

```text
/data1/jinyu_wang/projects/PILOT-Dub
```

The original `Glo-Var` workspace was not modified.

## 2. Kept Scope

The migrated project keeps only the final PILOT-Dub path:

```text
frozen generator -> K candidates -> WavLM-AVSync scorer -> selected output
```

Included:

- public generator configs: `pilot-dub/base` and `pilot-dub/strong`
- minimal inherited generator config chain needed by Hydra
- WavLM-AVSync scorer training and selection code
- LRS3 inference/evaluation wrappers
- final notes/reports
- final physical model artifacts

Excluded:

- old run scripts for v8-v15 exploratory branches
- temporary probes
- old training loops unrelated to the final method
- pycache / backup files

## 3. Model Artifacts

Physical model files are stored under:

```text
artifacts/PILOT-Dub/
```

Included:

```text
generator_base_epoch00.ckpt
generator_strong_epoch00.ckpt
avsync_scorer_best.pt
avsync_scorer_last.pt
avsync_scorer_resume_step=6000.pt
avsync_scorer_wandb_id.txt
```

Required pretrained files are physical copies:

```text
artifacts/pretrained_models/tokenizers/encodec.th
artifacts/pretrained_models/large_lrs3_iter5.pt
artifacts/torch/hub/checkpoints/wavlm_base_plus.pth
```

## 4. Data

Dataset files are symlinked:

```text
data/dataset -> /data1/jinyu_wang/projects/Glo-Var/data/dataset
```

This avoids duplicating LRS3.

## 5. Validation

The migrated project passed:

- shell syntax checks for final scripts
- Python compile checks for inference/scorer/model files
- `pilot-dub/strong` mini20 single-sample generation smoke test
- WavLM-AVSync selection smoke test on existing mini20 candidates
