# PILOT-Dub 最终整合版本笔记

## 0. 版本定位

当前项目只保留一个最终方法：

```text
PILOT-Dub = planner-aware visual TTS generator
            + multi-candidate generation
            + independent WavLM-AVSync reranking
```

项目中不再使用实验版本号作为对外命名。公开模型入口分成两个：

| 名称 | Hydra config | 用途 |
|---|---|---|
| base | `pilot-dub/base` | clean paper-base generator |
| strong | `pilot-dub/strong` | default empirical strong generator |

其中：

- `base` 继承 v14.5 的 clean hierarchical generator 路线；
- `strong` 继承 v14.6b 的 stronger generator 路线；
- 最终系统默认用 `strong + AVSync reranker`。

---

## 1. 核心目标

PILOT-Dub 的目标是：

> 在不破坏 frozen generator 的自然度和 WER 稳定性的前提下，通过逐样本候选选择提升音视频同步。

指标目标：

- 降低 `LSE-D`
- 提高 `LSE-C`
- 保持 `WER`
- 保持 `UTMOS`
- 保持 speaker similarity

---

## 2. 完整系统流程

```text
Text + reference audio + target video
        ↓
Frozen generator
        ↓
K stochastic dubbing candidates
        ↓
Independent WavLM-AVSync scorer
        ↓
Best candidate selection
        ↓
Final dubbed output
```

### 2.1 Generator

默认 strong generator：

```text
config: pilot-dub/strong
ckpt: artifacts/PILOT-Dub/generator_strong_epoch00.ckpt
```

clean base generator：

```text
config: pilot-dub/base
ckpt: artifacts/PILOT-Dub/generator_base_epoch00.ckpt
```

### 2.2 Candidate Generation

默认生成三个候选：

| candidate | seed | top_p | temperature |
|---|---:|---:|---:|
| cand0 | 1 | 0.8 | 1.0 |
| cand1 | 2 | 0.8 | 1.0 |
| cand2 | 3 | 0.9 | 0.9 |

### 2.3 Scorer

scorer checkpoint：

```text
artifacts/PILOT-Dub/avsync_scorer_best.pt
```

scorer 输入：

```text
candidate wav + target AV-HuBERT lip feature
```

scorer 输出：

```text
sync margin score
```

score 定义：

$$
s =
p(\Delta=0)
-
\max_{\Delta \neq 0}p(\Delta)
$$

---

## 3. 为什么最终项目分成 base 和 strong

### base

`base` 更适合论文中的 clean architecture narrative：

- delayed-codebook aware；
- hierarchical residual conditioning；
- 避免过多复杂 planner head；
- 结构更容易解释。

### strong

`strong` 更适合最终指标系统：

- 继承 v14.6b 的 stronger generator；
- 作为默认 practical system；
- 用于最终主表。

最终论文可以写成：

```text
PILOT-Dub is generator-agnostic.
We report both base and strong generator variants.
The strongest system uses the strong generator with AVSync reranking.
```

## 6. 一句话总结

PILOT-Dub 的最终命名不再是某个实验版本，而是：

> 一个具有 base / strong 两个 generator 变体的 planner-aware visual dubbing generator-reranker 框架。

核心贡献是：

```text
planner-aware candidate generation
    +
independent WavLM-AVSync candidate selection
```
