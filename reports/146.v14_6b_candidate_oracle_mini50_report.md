# v14.6b Candidate Oracle Mini50 Follow-up Report

## 0. 结论摘要

mini50 的结果把问题进一步说清楚了：

- 固定的全局 sampling 配置不稳定。`cand1 seed2` 在 mini20 看起来好，但 mini50 上全面变差。
- `cand2 t0.9/p0.9` 也不是稳定全局改进，mini50 上 WER、LSE-D、LSE-C 都不如 default。
- 但是，`cand0/cand1/cand2` 的 **WER-constrained oracle** 在 mini50 上仍然非常强，说明好候选确实存在，不是 mini20 小样本偶然。
- 当前真正缺失的是一个 **不使用最终 LSE 指标的 candidate scorer**。

## 4. 当前最精确结论

### 4.1 已经证明的内容

1. frozen `v14.6b` 的候选空间里存在明显更好的同步候选。
2. 这个结论在 mini20 和 mini50 都成立。
3. 单一 sampling seed/top-p/temp 不能稳定改进。
4. WER/UTMOS/SPK 都不是可靠的 sync selector。

### 4.2 还没有解决的内容

真正缺的是：

> paper-safe independent AV sync scorer。

不能直接用最终 LSE/SyncNet 选候选，否则是 metric hacking。需要训练或构建一个独立于最终 LSE evaluator 的 scorer。

---

## 5. 下一步建议

下一步应该暂停模型结构修改，转向 scorer：

1. 训练一个轻量 AV sync scorer：
   - 正样本：真实 LRS3 train audio-video pair；
   - 负样本：同视频配 shifted audio，或同 speaker/different utterance audio；
   - 输入：audio envelope / HuBERT-or-EnCodec acoustic feature + AV-HuBERT lip feature；
   - 输出：sync score 或 offset classification。

2. 用 mini50 的 `cand0/cand1/cand2` 作为 held-out candidate reranking test：
   - scorer 不能看最终 LSE；
   - 目标是恢复 oracle LSE gain 的至少一半；
   - WER 不能变差。

3. 如果 independent scorer 也失败：
   - candidate reranking 只能作为 oracle diagnostic；
   - 论文主线应回到 `v14.6b` best baseline + negative findings；
   - 不应继续做 q0/planner 局部控制。

当前最有希望的下一版不是 `v15.x` 模型结构，而是：

> `Rerank-1`: Frozen v14.6b + independently trained AV sync scorer + WER/quality guard。
