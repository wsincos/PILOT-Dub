# Independent AV Sync Scorer Pilot Failure Report

## 0. 目的

前面 mini20/mini50 candidate oracle 证明：

> frozen `v14.6b` 的候选空间里存在明显更好的同步候选。

但直接用最终 LSE/SyncNet 选候选是 oracle，只能证明潜力，不能作为方法。因此本轮实现一个独立 scorer pilot：

- 不使用最终 LSE 标签；
- 只使用真实 LRS3 train audio-video 对；
- 通过人工 temporal shift 构造 offset classification；
- 再用该 scorer 对 mini50 的 `cand0/cand1/cand2` 做候选选择。

---

## 1. Scorer 设计

### 1.1 输入

- Video: 已有 AV-HuBERT lip feature，shape 约为：

$$
V \in \mathbb{R}^{T \times 1024}
$$

- Audio: 从真实 mp4 / generated wav 中提取 16kHz waveform，再构造近似 log-mel：

$$
A \in \mathbb{R}^{T \times 80}
$$

其中 audio 先以 100Hz 提取，再平均池化到 25Hz。

### 1.2 训练任务

offset classes:

```text
[-8, -4, -2, 0, 2, 4, 8]
```

对每个真实 train clip：

1. 取一个 video window；
2. 取一个 shifted audio window；
3. 让模型预测 offset class。

目标：

$$
\mathcal{L}_{sync} = CE(f(A_{t+\Delta}, V_t), \Delta)
$$

### 1.3 模型

轻量结构：

- audio encoder: `LayerNorm -> Linear -> GELU -> Linear`
- video encoder: `LayerNorm -> Linear -> GELU -> Linear`
- fusion: `[a, v, |a-v|, a*v]`
- temporal encoder: 2-layer `Conv1d`
- head: offset classification

实现文件：

- `scripts/train_avsync_scorer_pilot.py`
- `scripts/rerank_candidates_with_avsync_scorer.py`
- `scripts/run_avsync_scorer_pilot_and_rerank.sh`

---

## 2. 训练设置

为了快速 pilot：

```bash
bash scripts/run_avsync_scorer_pilot_and_rerank.sh 0
```

实际训练参数：

- train samples: `1000`
- val samples: `200`
- epochs: `2`
- window: `32` frames at 25Hz
- batch size: `64`
- GPU: `0`

训练输出：

```text
epoch=0 step=0 train_loss=1.9495
{"epoch": 0, "loss": 1.9422, "acc": 0.2000, "zero_acc": 0.1250}
saved artifacts/Glo-Varv15/avsync_scorer_pilot.pt
epoch=1 step=0 train_loss=1.9371
{"epoch": 1, "loss": 1.9420, "acc": 0.1500, "zero_acc": 0.0000}
```

解释：

- 7-class random accuracy 约为 `1/7 = 14.3%`。
- 当前 val acc 只有 `15-20%`，基本接近随机。
- zero-offset accuracy 也非常低。

这说明当前 scorer 没有真正学会 AV temporal offset。

## 4. 失败原因

### 4.1 特征太弱

当前 audio 特征只是轻量 log-mel，video 特征是 AV-HuBERT frame feature，但模型没有使用强语义/phonetic representation。

这导致它更像在学习粗粒度 energy/activity，而不是 lip-speech phoneme alignment。

### 4.2 训练任务太小

`1000 train / 200 val` 只够 smoke test，不够训练一个可靠 scorer。

但是从 val accuracy 接近随机来看，即使扩大数据，当前结构也未必高效。

### 4.3 window-level offset classification 不等价于 candidate-level sync

真实 pair 的 shift 分类和 generated candidate reranking 存在 domain gap：

- 真实 audio 清晰、自然；
- generated audio 有 codec artifact、发音错误、停顿异常；
- scorer 在真实 shifted pair 上学到的模式未必能迁移到 generated candidate。

---

## 5. 当前决策

这个 pilot 没有通过成功标准：

- validation offset accuracy 没有超过 `70%`；
- mini50 rerank 没有改善 LSE；
- WER 还变差。

因此当前 scorer 不能继续作为方法。

但是 candidate oracle 仍然成立，所以最准确结论是：

> 候选空间有价值，但我们还没有找到可用 selector。

---

## 6. 下一步建议

如果继续 scorer 路线，必须提高 scorer 强度，而不是继续当前轻量版本：

1. 使用真正的 pretrained audio representation：
   - wav2vec2 / HuBERT / Whisper encoder features；
   - 不要只用 log-mel。

2. 使用更接近 SyncNet 的 contrastive training：
   - audio-video embedding；
   - offset classification；
   - hard negatives。

3. 使用 generated-domain adaptation，但不能用最终 LSE oracle label：
   - generated audio + artificial shifts；
   - same video, same generated audio shifted as negatives；
   - 不直接标注“哪个 candidate 最好”。

4. 如果没有时间实现强 scorer：
   - 暂停 reranking 方法；
   - 将 candidate oracle 作为 analysis；
   - 论文主线回到 `v14.6b` best baseline 与负结果分析。

当前不建议继续：

- q0/planner adapter；
- q0 token prior；
- activity-envelope scorer；
- 仅靠 global sampling config。
